## Rules being applied (from `.claude/skills/sigma-workbook-conventions/reference/workbook-spec-api.md`)

1. Every column referenced downstream is declared explicitly on the producing
   element with a stable, readable id (`col-date`, `col-store-region`, etc.)
   so controls can bind to it cleanly.
2. KPIs / chart / controls all source from the table element (sibling-element
   pattern); each redeclares the columns it needs.
3. KPI value formulas use `[Metrics/<Name>]` rather than redoing the math.
4. Control `filters[].columnId` references the table's column ids
   (e.g. `col-date`), NOT column display names.
5. Inside calc formulas, bare `[ColName]` resolves to the element's own
   columns by display name; passthrough columns omit the `name` field so they
   default to the source column's name.
6. Always visually verify after CREATE — the API doesn't catch broken
   cross-element column resolution.
